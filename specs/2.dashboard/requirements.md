# Dashboard — 需求规格

## 概述

控制面板页面，展示系统 KPI、活跃流水线列表和模型运行时状态。

## 项目信息

- 项目名: x-workflow
- 架构类型: monorepo (Next.js + Hono/tRPC)

## 需求版本

| 日期       | 版本 | 说明             |
| ---------- | ---- | ---------------- |
| 2026-06-18 | v1   | 初始需求         |

## 用户故事

- 作为用户，我希望在登录后看到系统概览，以便快速了解当前运行状态。
- 作为用户，我希望查看活跃的 AI 流水线列表，以便选择继续操作。
- 作为用户，我希望看到 AI 模型的运行性能指标，以便评估系统负载。

## 功能需求

1. [F-001] 欢迎横幅 — 显示系统运行状态徽章、欢迎文案、操作入口按钮
2. [F-002] KPI 卡片 — 展示总流水线条数、活动节点数、系统负载（三个卡片）
3. [F-003] 活跃流水线列表 — 显示最近运行的流水线：ID、名称、节点数、状态、时间
4. [F-004] 模型运行时卡片 — 展示加载的 AI 模型：名称、延迟、可靠性、使用量
5. [F-005] 流水线状态徽章 — Active 显示绿色，Paused 显示灰色
6. [F-006] 跳转入口 — 点击"进入画布"跳转到编排器，点击"分析堆栈"跳转到调试器

## 非功能需求

- 性能: 仪表盘首屏加载 < 1.5s
- 安全: 数据仅显示当前用户有权限访问的流水线
- 响应式: 移动端适配，卡片堆叠显示

## 验收标准

- [ ] [AC-001] 登录后自动跳转到 /dashboard 并显示欢迎横幅
- [ ] [AC-002] KPI 卡片正确显示统计数据（从后端 API 获取）
- [ ] [AC-003] 流水线列表显示最近运行的 5 条记录
- [ ] [AC-004] 点击"进入画布"成功跳转到 /orchestrator
- [ ] [AC-005] 模型运行时显示延迟和可靠性百分比
