# Orchestrator Canvas — 需求规格

## 概述

可视化 AI 编排器，支持节点库、画布拖拽、节点配置、工作流持久化。

## 项目信息

- 项目名: x-workflow
- 架构类型: monorepo (Next.js + Hono/tRPC)

## 需求版本

| 日期       | 版本 | 说明             |
| ---------- | ---- | ---------------- |
| 2026-06-18 | v1   | 初始需求         |

## 用户故事

- 作为用户，我希望从节点库拖拽节点到画布，以便构建 AI 处理流水线。
- 作为用户，我希望配置节点的参数（URL、模型、提示词等），以便自定义节点行为。
- 作为用户，我希望保存和加载工作流，以便下次继续编辑。

## 功能需求

1. [F-001] 节点库面板 — 左侧展示 5 种节点类型：Trigger、Condition、Parallel、Multi-Modal、LLM Synthesis
2. [F-002] 可视化画布 — 中央显示节点卡片，带 SVG 连接线，支持节点选择
3. [F-003] 节点配置面板 — 右侧根据选中节点类型显示对应配置项
4. [F-004] 节点类型配置 — Trigger（URL）、LLM Synthesis（模型、系统指令、temperature）、全局（maxRetries、timeout、haltOnError）
5. [F-005] 工作流保存 — 保存当前画布状态到数据库
6. [F-006] 工作流加载 — 从数据库加载已保存的工作流
7. [F-007] 新建/重置工作流 — 创建空白画布或恢复到默认预设
8. [F-008] 节点增删 — 点击节点库添加节点，点击删除按钮移除节点

## 非功能需求

- 性能: 节点拖拽流畅，60fps
- 安全: 工作流数据按用户隔离
- 存储: 节点位置、配置序列化存储

## 验收标准

- [ ] [AC-001] 点击节点库项可在画布添加新节点
- [ ] [AC-002] 点击画布节点可选择并显示配置面板
- [ ] [AC-003] 修改节点配置后保存成功
- [ ] [AC-004] 刷新页面后工作流状态保持
- [ ] [AC-005] 可创建新工作流并保存
