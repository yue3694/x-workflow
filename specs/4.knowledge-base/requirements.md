# Knowledge Base — 需求规格

## 概述

知识库文档管理，支持文件上传、搜索过滤、RAG 向量化。

## 项目信息

- 项目名: x-workflow
- 架构类型: monorepo (Next.js + Hono/tRPC)

## 需求版本

| 日期       | 版本 | 说明             |
| ---------- | ---- | ---------------- |
| 2026-06-18 | v1   | 初始需求         |

## 用户故事

- 作为用户，我希望上传文档到知识库，以便为 AI 流水线提供上下文知识。
- 作为用户，我希望搜索和过滤文档，以便快速找到需要的资料。
- 作为用户，我希望系统自动对文档进行向量化处理，以便支持语义检索。

## 功能需求

1. [F-001] 文档表格 — 展示已上传文档：ID、名称、类型、大小、上传时间
2. [F-002] 搜索过滤 — 支持按文档名称和类型搜索过滤
3. [F-003] 拖拽上传 — 支持拖拽或点击选择文件上传（PDF、TXT、CSV、EPUB）
4. [F-004] 文件大小计算 — 自动计算并显示文件大小
5. [F-005] 文档删除 — 删除已上传的文档及其向量数据
6. [F-006] RAG 向量化 — 上传后自动调用 RAG pipeline 进行文本提取和向量化
7. [F-007] RAG 参数配置 — 显示向量模型、分块大小、压缩方式等参数
8. [F-008] 文档下载 — 支持下载已上传的原始文档

## 非功能需求

- 性能: 大文件上传支持进度显示
- 安全: 文件类型白名单校验（MIME type）
- 存储: 文件存储到服务器本地目录，向量存储到数据库

## 验收标准

- [ ] [AC-001] 可通过拖拽或点击上传文档到知识库
- [ ] [AC-002] 上传后文档自动出现在列表顶部
- [ ] [AC-003] 搜索框输入可实时过滤文档列表
- [ ] [AC-004] 点击删除按钮可删除文档及其向量数据
- [ ] [AC-005] RAG 参数面板正确显示配置信息
- [ ] [AC-006] 支持 PDF、TXT、CSV 格式上传

## 依赖

- 1.auth-login: 需要登录才能使用
