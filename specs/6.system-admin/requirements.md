# System Admin — 需求规格

## 概述

系统管理面板，支持用户目录、成员邀请、权限拓扑配置。

## 项目信息

- 项目名: x-workflow
- 架构类型: monorepo (Next.js + Hono/tRPC)

## 需求版本

| 日期       | 版本 | 说明             |
| ---------- | ---- | ---------------- |
| 2026-06-18 | v1   | 初始需求         |

## 用户故事

- 作为管理员，我希望查看所有工作空间用户，以便管理成员。
- 作为管理员，我希望邀请新成员加入工作空间，以便扩展团队。
- 作为管理员，我希望配置权限拓扑和安全策略，以便加强系统安全。

## 功能需求

1. [F-001] 用户目录表格 — 展示所有用户：头像、姓名、邮箱、角色、权限级别、最后活跃时间
2. [F-002] 角色标签 — ADMIN 显示红色，EDITOR 显示蓝色，VIEWER 显示灰色
3. [F-003] 删除用户 — 删除非管理员用户，显示确认提示
4. [F-004] 邀请成员弹窗 — 点击"添加成员"显示邀请表单
5. [F-005] 邀请表单 — 输入姓名、邮箱，选择角色和权限级别
6. [F-006] 权限拓扑面板 — 显示 3 个安全开关：Strict Hierarchy、Encryption Vault、External Access
7. [F-007] 权限开关状态保存 — 开关状态保存到数据库
8. [F-008] Pro 升级卡片 — 显示企业版功能介绍和升级按钮（UI 预留）

## 非功能需求

- 性能: 用户列表分页加载（每页 20 条）
- 安全: 仅 ADMIN 角色可访问系统管理
- 权限: 非管理员用户不能删除自己或访问管理功能

## 验收标准

- [ ] [AC-001] 管理员可查看所有用户列表
- [ ] [AC-002] 点击"添加成员"显示邀请表单
- [ ] [AC-003] 提交邀请表单后新用户出现在列表
- [ ] [AC-004] 点击删除按钮可删除用户
- [ ] [AC-005] 权限开关状态保存后刷新页面保持
- [ ] [AC-006] 非管理员访问 /admin 被重定向

## 依赖

- RBAC 实现: 依赖权限系统
